# Pacote de Venda - Checklist

## Pre-venda (antes da proposta)
1. Definir objetivo do cliente:
- reduzir risco, melhorar disciplina, ou monitor setorial.
2. Confirmar universo e setores.
3. Definir responsavel tecnico do cliente.
4. Definir horario de entrega diaria.

## Proposta
1. Escopo do piloto (30 dias).
2. Entregas diarias e semanais.
3. Limites honestos do produto:
- nao preve dia exato de crash
- nao promete retorno.
4. Criterio de sucesso objetivo.
5. Preco e prazo.

## Kickoff (Dia 0)
1. Alinhar regra de acao por nivel (`verde`, `amarelo`, `vermelho`).
2. Validar canais de envio (email, slack, etc.).
3. Validar formato do resumo semanal.
4. Registrar baseline inicial de risco/carteira.

## Operacao (Dias 1-30)
1. Rodar pipeline diario.
2. Enviar resumo diario curto.
3. Registrar eventos acionados.
4. Registrar decisoes do cliente.
5. Revisao semanal de utilidade.

## Fechamento (Dia 30)
1. Consolidar metricas do periodo.
2. Mostrar casos de uso reais no periodo.
3. Mostrar custo de falso alerta.
4. Recomendar:
- continuar mensal
- continuar com ajuste
- encerrar.

## Saidas minimas para apresentar
1. `docs/OFERTA_COMERCIAL_MOTOR.md`
2. `docs/MOTOR_PROVA_3P.md`
3. `docs/PILOTO_30D_PLAYBOOK.md`
4. `results/dual_mode_compare/summary_7d.json` (quando houver 7 dias)
